# 知乎 A2A 社交创作助手

> Agent-to-Agent 双向社交对话，精准对齐创作需求 · AI辅助 · 人为主导

## 一句话介绍

通过多智能体社交交互模拟真实用户与创作者的沟通场景，基于知乎真实数据为创作者输出**写作框架**，拒绝AI代写全文，解决答主创作内容与用户真实需求不匹配的问题。

---

## 核心创新点

### 1. 首创「社交主体 + 后台工具」双轨Agent架构

区别于传统AI工具的单向流水线，本项目构建4个分工明确的智能体，严格区分**真正参与社交对话的主体**与**静默执行的后台工具**：

| 类型 | Agent | 角色定位 |
|------|-------|---------|
| 🔵 社交主体（核心A2A） | 用户Agent | 代表真实提问用户，有情绪、会吐槽、会拒绝 |
| 🔵 社交主体（核心A2A） | 答主Agent | 代表知乎创作者，主动确认、灵活调整方案 |
| ⚙️ 后台工具 | 采集Agent | 静默抓取知乎真实问题、高赞回答、热榜数据 |
| ⚙️ 后台工具 | 过滤Agent | 剔除AI水文、广告、劝退型硬核内容 |

### 2. 真实双向社交对话（非单向问答）

- **用户Agent**：口语化表达、带情绪、会质疑、逐步补充需求，像真人一样「有来有回」
- **答主Agent**：专业但不傲慢，主动确认「我理解对吗？」、给出选择「思路A还是B？」
- **至少3轮**深度对话，逐步对齐需求、明确禁忌、达成共识

### 3. 严格遵循知乎价值观：AI辅助、人为主导

- ❌ **绝不生成完整回答正文** —— 只输出写作框架、切入角度、结构规划
- ❌ **抵制AI水文** —— 过滤Agent自动识别并剔除AI生成内容
- ✅ **创作者真人填充** —— 系统只提供脚手架，内容必须由真人创作

└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                     后端 (FastAPI + uvicorn)                 │
│  ┌─────────────────────────────────────────────────────────┐│
│  │              SessionOrchestrator (编排器)                ││
│  │  Step1: CollectorAgent + FilterAgent → 数据准备         ││
│  │  Step2: UserAgent ↔ AnswererAgent (3+轮对话)            ││
│  │  Step3: ConsensusBoard → 需求对齐看板                   ││
│  │  Step4: WriterFramework → 创作者写作框架                ││
│  └─────────────────────────────────────────────────────────┘│
│                         Anthropic Claude API                  │
└─────────────────────────────────────────────────────────────┘
                              │
                    知乎开放接口（热榜、搜索、内容）

### 技术栈

- **前端**：Vite 5 + React 18 + TypeScript，1:1复刻知乎原生UI（蓝白配色、卡片布局）
- **后端**：FastAPI + Python 3.11 + uvicorn，WebSocket实时推送全流程
- **LLM**：Anthropic Claude（claude-sonnet-4-6），通过 `pydantic-settings` 配置
- **数据源**：知乎热榜API、知乎搜索API
- **部署**：Docker 单容器（Nginx + uvicorn），支持 Sealos 云原生平台一键部署

## 4步交互流程（Demo演示路径）

### Step 1 — 数据采集与过滤（后台静默）

用户输入知乎问题链接 → 采集Agent抓取问题、高赞回答、热榜上下文 → 过滤Agent剔除AI水文/广告/硬核内容 → 保留高质量参考素材。

**可视化**：话题启动区展示采集Agent和过滤Agent的执行状态，进度一目了然。

### Step 2 — A2A多轮社交对话（核心亮点）

启动用户Agent与答主Agent的双向对话：
- 第1轮：用户表达核心需求（模糊但真实）→ 答主复述确认、给出初步思路
- 第2轮：用户提出顾虑/质疑 → 答主调整方向、细化方案
- 第3轮：用户明确底线和禁忌 → 答主给出精准创作框架

**可视化**：A2A实时社交日志面板，逐行动态展示对话，蓝色=用户Agent，绿色=答主Agent，标注时间戳和轮次。

### Step 3 — 需求对齐看板

从3轮对话中提取共识，生成4个可视化卡片：
- 🎯 用户核心需求
- 🚫 用户禁忌内容
- ✍️ 答主创作方向
- 💡 关键洞察

### Step 4 — 创作者写作框架（禁止全文）

基于共识输出创作脚手架：
- 📌 标题建议（多个备选）
- 🎯 切入角度（不同思路）
- 📐 结构大纲（可折叠层级）
- ⚠️ 避雷清单
- 🎭 语气风格指导

**绝对红线**：本步骤仅输出框架，不生成任何可直接发布的完整段落。

## 与知乎生态的契合度

| 知乎生态痛点 | 本方案解决方式 |
|-------------|--------------|
| 答主写完发现「文不对题」，用户真实需求没搞清 | A2A社交对话提前对齐需求 |
| 用户评论区吐槽「没回答到点子上」 | 需求对齐看板明确记录共识 |
| AI水文泛滥，社区质量下降 | 过滤Agent + 仅输出框架（不代写） |
| 创作者不知道从何下笔 | 写作框架提供结构化脚手架 |
| 平台希望赋能创作者而非替代 | 「AI辅助、人为主导」贯穿始终 |

## 快速开始

### 本地开发

```bash
# 1. 克隆仓库
git clone <repo-url>
cd A2Aforzhihu

# 2. 后端
cd backend
cp .env.example .env
# 编辑 .env，填入 ANTHROPIC_API_KEY
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py

# 3. 前端（新终端）
cd frontend
npm install
npm run dev
# 访问 http://localhost:5173
```

### 线上部署（Sealos）

```bash
# 1. 构建镜像
docker build -t a2a-zhihu .

# 2. 推送到 Docker Hub
docker tag a2a-zhihu your-dockerhub-username/a2a-zhihu
docker push your-dockerhub-username/a2a-zhihu

# 3. Sealos 创建应用，配置环境变量
# ANTHROPIC_API_KEY=sk-xxx
```
